# Feature Prioritization

## Overview

This document prioritizes Smart NUB Campus features using the MoSCoW method.

---

# Must Have

These features are essential for the first release.

- Student Registration
- Student Verification
- Authentication
- Student Profiles
- Academic Resource Sharing
- Resource Search
- Discussion Forum
- Question & Answer
- Student Network
- Connection Requests
- Private Messaging
- Team Finder
- AI Academic Assistant
- Admin Dashboard
- Notifications

---

# Should Have

These features add significant value but are not required for the first release.

- Reputation System
- Badges
- Resource Bookmarking
- Advanced Search Filters
- Student Recommendations
- Team Recommendations

---

# Could Have

These features may be implemented if time permits.

- OCR Student ID Verification
- AI-assisted Verification
- AI Resource Recommendations
- AI-generated Weekly Study Plans
- Faculty Accounts
- Alumni Accounts

---

# Won't Have (Version 1)

These features are intentionally excluded.

- Course Registration
- Tuition Payment
- Attendance Tracking
- Result Publishing
- Faculty Management
- Official ERP Functions

---

# Review Strategy

Priorities should be reviewed before each major release.
