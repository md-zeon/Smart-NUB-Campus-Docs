# Non-Functional Requirements

## Overview

This document defines the quality attributes and constraints of Smart NUB Campus (SNC).

Unlike functional requirements, non-functional requirements specify how the system should behave in terms of performance, usability, security, scalability, reliability, and maintainability.

---

# Performance

## NFR-PERF-001

The system should load most pages within 2 seconds under normal usage.

## NFR-PERF-002

Search results should be returned within 2 seconds.

## NFR-PERF-003

Private messages should be delivered in near real time.

## NFR-PERF-004

The AI assistant should respond within a reasonable time depending on request complexity.

---

# Security

## NFR-SEC-001

Only authenticated and verified students may access protected resources.

## NFR-SEC-002

Passwords must never be stored in plain text.

## NFR-SEC-003

Sensitive communication must be encrypted during transmission.

## NFR-SEC-004

Uploaded files must be validated before storage.

## NFR-SEC-005

Administrative actions should be logged for auditing.

---

# Reliability

## NFR-REL-001

The system should recover gracefully from unexpected failures.

## NFR-REL-002

Data integrity must be preserved during failures.

## NFR-REL-003

Critical user data should be backed up regularly.

---

# Scalability

## NFR-SCL-001

The system should support increasing numbers of students without major architectural changes.

## NFR-SCL-002

The architecture should support future expansion to additional university modules.

---

# Availability

## NFR-AVL-001

The platform should be available whenever students need academic resources, except during scheduled maintenance.

---

# Usability

## NFR-USE-001

The interface should be simple and intuitive.

## NFR-USE-002

Students should be able to complete registration without external guidance.

## NFR-USE-003

The platform should minimize distractions and keep users focused on academics.

---

# Maintainability

## NFR-MNT-001

The codebase should follow a modular architecture.

## NFR-MNT-002

System modules should be independently maintainable.

## NFR-MNT-003

All APIs should be documented.

---

# Compatibility

## NFR-CMP-001

The platform should support modern desktop and mobile browsers.

## NFR-CMP-002

The interface should be responsive across common screen sizes.

---

# AI

## NFR-AI-001

AI responses should clearly indicate that they are generated by AI.

## NFR-AI-002

The AI assistant should prioritize educational guidance over direct answers when appropriate.
